# 🔷 4. Authentication System

Use custom user model.

User model must include:

* email (unique, login field)
* name
* phone
* role (USER, MANAGER, ADMIN)
* is_verified

Use JWT authentication with SimpleJWT.

Use Djoser for:

* Registration
* Email activation
* Password reset

JWT header type must be:

```
Authorization: JWT <token>
```

---

# 🔷 5. Role System

Three roles:

1. USER
2. MANAGER
3. ADMIN

Permissions:

* USER → book restaurants
* MANAGER → manage own restaurants
* ADMIN → full control

Create custom DRF permission classes:

* IsAdmin
* IsManager
* IsUser

---

# 🔷 6. Apps Responsibilities

---

## users app

* Custom User model
* Role-based permissions
* Serializers
* Signals if needed

---

## restaurants app

Models:

* Restaurant
* Branch
* Table
* MenuItem

Features:

* Restaurant has manager (FK to User)
* Branch belongs to Restaurant
* Table belongs to Branch
* Table fields:

  * table_number
  * seat_capacity
  * seat_type (W, C, NORMAL)
  * is_reserved
* MenuItem belongs to Restaurant

---

## bookings app

Models:

* Booking
* BookingMenu

Booking fields:

* booking_id (RSX-2026-000001 format)
* user
* restaurant
* branch
* table
* date
* start_time
* duration (1 or 2 hours max)
* status
* total_price
* created_at

Booking statuses:

```
PENDING_PAYMENT
CONFIRMED
REJECTED
CANCELLED
EXPIRED
COMPLETED
```

BookingMenu:

* booking
* menu_item
* quantity

Add unique booking ID generator.

Prevent double booking of same table + same time.

---

## payments app

Model:

* Payment

  * booking
  * amount
  * payment_status
  * transaction_id
  * created_at

Create endpoints:

```
/api/v1/payments/start/<booking_id>/
/api/v1/payments/success/
/api/v1/payments/fail/
```

On success:

* Booking → CONFIRMED
* Table → reserved

On fail:

* Booking → PENDING_PAYMENT
* Auto expire after 1 minute (simple implementation)

---

## dashboard app

User Dashboard:

* Total bookings
* Favorite restaurant (most booked)
* Booking history
* Booking status list

Manager Dashboard:

* All bookings for owned restaurants
* Confirm / Reject booking
* Add / remove tables
* Modify menu
* Add branch
* Assign waiter name per table

Admin Dashboard:

* Change user roles
* Remove manager
* Remove restaurant
* If restaurant has no manager → delete restaurant automatically

Use signals to auto delete restaurant if manager removed.
